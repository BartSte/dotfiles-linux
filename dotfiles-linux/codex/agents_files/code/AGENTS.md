# Commit Messages

- Prefix commit subjects using Conventional Commits: `<type>[optional scope]: <description>`.
- Use common types such as `feat`, `fix`, `docs`, `refactor`, `test`, `chore`, `build`, `ci`, `style`, and `perf`.
- Add a scope when it clarifies the affected area, for example `docs(codex): add commit guidance`.
- Write the description in imperative mood, lowercase after the prefix unless a proper noun requires capitalization, and omit the trailing period.
- Keep the subject concise. If more context is needed, add a blank line and explain the why in the body.
